# Wise Homeopathy GitHub Workflow
## Version 2.0

## Branch Strategy
main -> Production
develop -> Integration
feature/<name> -> New work
fix/<name> -> Bug fixes

## Workflow
1. Create feature branch.
2. Review documentation before coding.
3. Make focused commits.
4. Open Pull Request.
5. Review.
6. Merge into develop.
7. Test.
8. Merge to main.
9. Deploy.

## Commit Style
feat:
fix:
refactor:
docs:
style:
perf:
chore:

Example:
feat: improve mobile navigation
fix: correct canonical URLs
docs: update design system

## Pull Request Checklist
- Documentation reviewed
- No broken links
- Mobile tested
- Accessibility checked
- SEO verified
- Screenshots attached (if UI changed)
