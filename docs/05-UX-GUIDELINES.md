# Wise Homeopathy UX Guidelines
## Version 2.0

> Defines the user experience principles for every page, interaction, and future feature.

# UX Philosophy

The website should feel effortless.

Patients should never have to think:
- Where am I?
- What do I do next?
- Where do I click?

The interface should answer those questions naturally.

---

# UX Principles

1. Patient First
2. Simplicity Before Features
3. Trust Before Conversion
4. One Primary Action Per Screen
5. Consistent Navigation
6. Mobile First
7. Accessibility by Default

---

# User Journey

Visitor

↓

Learns

↓

Builds Trust

↓

Books Consultation

↓

Becomes Patient

↓

My Health Journey (Future)

---

# Navigation Rules

Primary navigation:

- Home
- Conditions
- Online Consultation
- Doctors
- Resources
- Contact

Patient Login is always separate.

Every page must provide a logical next step.

No dead ends.

---

# Content Hierarchy

1. Short answer
2. Simple explanation
3. Detailed information
4. FAQ
5. Call to Action

Avoid long uninterrupted text.

---

# CTA Guidelines

Every page has ONE primary CTA.

Examples:
- Book Consultation
- Start Online Consultation
- Meet Our Doctors
- Read Related Article

Avoid multiple competing CTAs.

---

# Mobile UX

- Thumb-friendly buttons
- Sticky primary CTA where appropriate
- Floating Page Guide on long pages
- Responsive forms
- Fast loading

---

# Floating Page Guide

Long pages should include:

- Overview
- Symptoms
- Treatment
- FAQ
- Research
- Book Consultation

Highlight current section while scrolling.

---

# Forms

- Few fields
- Clear labels
- Helpful validation
- Confirmation after submission
- Never lose entered data

---

# Reading Experience

- Maximum 4-line paragraphs
- Clear headings
- Bullets over dense text
- Tables where helpful
- Timelines for journeys

---

# Trust Elements

Distribute trust throughout the page:

- Credentials
- Research
- Experience
- Clinic photos
- Transparent process

---

# Accessibility

- Keyboard navigation
- Visible focus states
- Semantic HTML
- Alt text
- WCAG AA contrast

---

# Performance

- Minimal JavaScript
- Optimized images
- Lazy loading
- Lighthouse target 95+

---

# Future UX

Future patient features must follow the same principles:

- My Health Journey
- Personal Care Plan
- Wise Digital Twin
- Symptom Tracker

They should feel like a natural continuation of the public website.

---

# Guiding Question

Before shipping any feature ask:

Does this make the patient's journey easier, clearer, and more reassuring?
